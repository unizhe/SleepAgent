# PROJECT_PLAN

## 项目定位

SleepAgent 是一个基于 SHHS 多导睡眠数据的智能睡眠健康分析系统。系统面向老年用户、家属和医生，目标是在保证医学表述谨慎的前提下，提供睡眠分期、呼吸暂停风险识别、趋势分析、可解释报告和对话式健康建议。

## 总体架构

系统分为五层：

1. 数据层
   - SHHS PSG 原始数据。
   - 睡眠分期标注。
   - 呼吸事件标注。
   - 用户基础信息、分析结果和长期历史记录。

2. 模型层
   - YASA 睡眠分期模型。
   - 自建 1D-CNN + BiLSTM 呼吸事件检测模型。
   - 指标评估模块。

3. 服务层
   - FastAPI 后端服务。
   - 数据管理服务。
   - 报警推送服务。
   - 外部数据工具服务。

4. Agent 层
   - 睡眠分析 Agent。
   - 报告生成 Agent。
   - 对话交互 Agent。
   - LangGraph 负责流程编排和状态管理。

5. 展示层
   - Streamlit 前端。
   - 睡眠总览、风险等级、趋势图、报告和问答入口。

## 数据流

1. 用户或实验流程选择一条 SHHS PSG 记录。
2. 数据管理服务读取信号、元数据和标注。
3. 睡眠分析 Agent 调用睡眠分期模型和呼吸事件检测模型。
4. 评估模块输出 Accuracy、Kappa、F1、AUC、Recall 等指标。
5. 睡眠分析 Agent 汇总 AHI、疑似呼吸暂停次数、呼吸频率趋势和风险等级。
6. 报告生成 Agent 从医学知识库检索相关说明，生成两个版本报告。
7. 对话交互 Agent 基于分析结果、用户画像和长期记忆回答问题。
8. Streamlit 展示核心结果和趋势图。
9. 报警推送服务在高危规则触发时生成预警。

## 模型计划

### 睡眠分期

- 依赖 YASA。
- 任务简化为 Wake、REM、NREM 三分类。
- 需要将原始睡眠阶段映射为三类：
  - Wake：Wake。
  - REM：REM。
  - NREM：N1、N2、N3、N4 或等价非 REM 阶段。
- 评估指标：
  - Accuracy。
  - Cohen's Kappa。
  - macro F1。
  - weighted F1。
  - Wake、REM、NREM 各类别 F1。

### 呼吸暂停检测

- 模型结构：1D-CNN + BiLSTM。
- 分类目标：
  - normal breathing。
  - hypopnea。
  - suspected apnea。
- 核心指标：
  - Recall。
  - AUC。
  - F1。
- 阶段目标：
  - AUC > 0.85。
  - Recall > 0.80。
- MVP 阶段优先定义数据窗口、标签映射、模型输入输出和评估脚本，不强行完成最终性能目标。

## Agent 计划

### 睡眠分析 Agent

职责：

- 调用睡眠分期模型。
- 调用呼吸事件检测模型。
- 计算睡眠总览、AHI、异常事件统计和风险等级。
- 将模型结果转为报告生成 Agent 可消费的结构化摘要。

### 报告生成 Agent

职责：

- 从 Chroma 医学知识库检索睡眠、呼吸暂停、老年健康相关知识。
- 生成老人易懂版报告。
- 生成子女/医生专业版报告。
- 给出谨慎的进一步就医建议。

### 对话交互 Agent

职责：

- 支持多轮问答。
- 结合用户历史结果给出个性化建议。
- 支持主动关怀问候。
- 对医疗高风险问题保持边界，提示就医而非替代诊断。

## 独立服务计划

### 报警推送服务

- 根据高危规则生成预警。
- 初期只记录预警事件。
- 后续扩展短信、邮件或应用推送。

### 数据管理服务

- 管理用户信息、分析结果、报告和长期历史。
- 管理长期记忆压缩。
- MVP 可先使用本地文件或轻量结构，后续迁移 PostgreSQL。

### 外部数据工具服务

- 接入天气、温度、饮食等生活方式因素。
- MVP 阶段先定义接口和 mock 数据。
- 后续再接入真实 API。

## 测试策略

- 每个模块都提供最小可运行测试。
- 数据映射和指标计算优先单元测试。
- 模型训练先做 smoke test，再做正式实验。
- Agent 输出使用结构化字段校验。
- 任何任务完成后更新 `TASK_LOG.md`。

## 风险与约束

- SHHS 数据访问需要授权，不能将敏感数据提交到项目中。
- 医疗建议必须谨慎，避免诊断性断言。
- YASA 与 SHHS 信号通道可能需要适配。
- 呼吸事件标签质量和窗口设计会显著影响 Recall。
- MVP 阶段应控制范围，避免同时推进模型、Agent、前端和部署。
