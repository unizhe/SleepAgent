# AGENTS

## Agent 总览

SleepAgent 计划包含三个核心 Agent：

- 睡眠分析 Agent。
- 报告生成 Agent。
- 对话交互 Agent。

三个 Agent 的设计原则是职责清晰、输入输出结构化、医学表述谨慎，并能在 MVP 阶段先用规则或模板实现，后续再逐步接入 LangGraph、RAG 和真实模型服务。

## 睡眠分析 Agent

### 职责

- 接收单条睡眠记录或分析任务。
- 调用 YASA 睡眠分期流程。
- 调用 1D-CNN + BiLSTM 呼吸事件检测流程。
- 汇总睡眠统计、呼吸事件统计和风险等级。
- 生成结构化分析摘要，供报告生成 Agent 和前端使用。

### 输入

- 用户或实验对象 ID。
- PSG 记录 ID。
- 信号路径或数据引用。
- 睡眠阶段标注，可选。
- 呼吸事件标注，可选。
- 用户基础信息，可选。

### 输出

- 睡眠分期结果。
- 睡眠总时长、Wake 时长、REM 时长、NREM 时长。
- AHI 指数。
- normal breathing、hypopnea、suspected apnea 统计。
- 呼吸频率趋势。
- 风险等级。
- 模型评估指标。
- 异常事件摘要。

### MVP 实现方式

- 先定义结构化输入输出。
- 可使用 mock 数据验证完整流程。
- 接入 YASA 后替换睡眠分期占位逻辑。
- 呼吸事件检测先保留模型接口，再逐步接入 PyTorch 模型。

## 报告生成 Agent

### 职责

- 接收睡眠分析 Agent 的结构化摘要。
- 检索医学知识库中的相关说明。
- 生成老人易懂版报告。
- 生成子女/医生专业版报告。
- 给出进一步就医建议。

### 输入

- 睡眠分析摘要。
- 用户年龄、性别、基础疾病等背景信息，可选。
- 检索到的医学知识片段。
- 报告风格参数。

### 输出

- 老人易懂版报告。
- 子女/医生专业版报告。
- 风险解释。
- 就医建议。
- 需要继续观察的指标。

### 医学表达边界

- 使用“提示”“可能”“建议进一步检查”等谨慎措辞。
- 不给出确定诊断。
- 不替代医生判断。
- 对胸痛、严重呼吸困难、意识异常等高危情况，应建议及时就医或急诊处理。

### MVP 实现方式

- 先使用模板生成报告。
- 将 RAG 检索接口设计为可替换组件。
- 后续接入 Chroma 和医学知识库。

## 对话交互 Agent

### 职责

- 支持用户围绕睡眠报告进行多轮问答。
- 将专业指标解释为老人和家属能理解的语言。
- 基于长期睡眠趋势给出生活方式建议。
- 主动进行关怀式提醒。

### 输入

- 用户问题。
- 当前睡眠分析结果。
- 历史睡眠摘要。
- 用户偏好和长期记忆。
- 外部生活方式数据，可选。

### 输出

- 问答回复。
- 个性化建议。
- 主动关怀问候。
- 必要时触发报告生成或报警推送。

### MVP 实现方式

- 先基于单次分析结果回答固定问题。
- 长期记忆先使用压缩摘要字段。
- 后续接入数据库和 LangGraph 状态管理。

## LangGraph 编排设想

MVP 之后，三个 Agent 可按以下流程编排：

1. `load_sleep_record`
2. `run_sleep_staging`
3. `run_respiratory_detection`
4. `summarize_analysis`
5. `assess_risk`
6. `retrieve_medical_knowledge`
7. `generate_reports`
8. `answer_user_question`
9. `maybe_trigger_alert`
10. `save_memory`

## 风险等级初步设想

风险等级需要结合 AHI、疑似呼吸暂停频率、血氧下降、睡眠结构异常和用户背景综合判断。MVP 阶段可以先使用简化规则，后续根据医学指南和实验结果迭代。

初步等级：

- 低风险：指标整体稳定，无明显异常趋势。
- 中风险：存在疑似呼吸暂停、睡眠结构紊乱或趋势恶化。
- 高风险：疑似呼吸暂停频繁、AHI 偏高或伴随明显异常提示。

## Agent 评估原则

- 睡眠分析 Agent 评估模型指标和统计正确性。
- 报告生成 Agent 评估事实一致性、医学安全性和可读性。
- 对话交互 Agent 评估多轮一致性、边界控制和建议可执行性。
